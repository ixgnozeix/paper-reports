1. 论文速览

1.1 一句话贡献

EgoScale 的核心是证明大规模人类第一视角数据可以作为灵巧手机器人学习的可预测监督源。它不是只做视频理解，而是训练 VLA 去预测 wrist motion 和 retargeted hand actions，再通过 aligned human-robot mid-training 把这种人类运动先验落到机器人动作空间。

在整篇论文中，这个要点对应 EgoScale 的主证据链：它必须同时说明任务为什么难、方法如何改变信息流、实验如何证明改变有效，以及哪些边界仍未解决。

阅读 EgoScale 时不要只看最终 demo，而要把贡献句拆成可验证问题：它改变了哪个中间表示，这个表示是否真的降低了接触、泛化或 sim-to-real 难度，实验是否覆盖了这种难度。

这也是判断论文价值的第一道筛选：如果贡献能被拆成清楚的输入、表示、输出和真实验证，那么它更适合后续复现和改造；如果只停留在概念层，就很难进入工程闭环。

1.2 技术定位

2. 研究问题与动机

2.1 核心问题

机器人遥操作数据昂贵，尤其是高 DoF 灵巧手；而人类每天产生大量第一视角操作视频。问题是这些视频没有机器人 proprioception 和可执行动作，能否变成真正有用的 robot motor prior。

这个问题定义决定了方法复杂度是否必要。若只是普通视觉模仿学习能够解决，触觉、空间锚定、人类 scaling 或物体中心 RL 都会显得多余；论文真正要证明的是旧表示在关键接触或泛化场景下不够。

对 EgoScale 来说，动机部分还要追问失败发生在感知、表示、数据还是控制接口。如果瓶颈判断错了，后面的架构再复杂也只是把错误包装得更精致。

因此动机不是背景介绍，而是后文实验设计的来源。一个好的实验应该正好打在这个瓶颈上，让我们能看出方法是否真的修复了原来的失败模式。

2.2 scaling 是否成立

4. 方法详解

4.1 两阶段框架

第一阶段在 20,854 小时人类 egocentric video 上预训练，监督 wrist motion 和 retargeted hand actions；第二阶段加入 aligned human-robot mid-training，将人类表示对齐到机器人 proprioception 和 action space。

从复现角度看，应把该模块的输入、输出和误差传播单独验收。灵巧手系统往往不是单点失败，而是表示误差、标定误差和控制误差在接触过程中连续放大。

EgoScale 的方法部分尤其要看中间变量是否可视化和可诊断：触觉图、空间编码、目标位姿、人类动作 token 或 deform map 一旦无法单独检查，真实部署失败时就很难定位原因。

实际实现时，每个模块都应配一个最小验收脚本：例如可视化触觉预测、检查坐标锚定、回放目标位姿、比较 sim-real deform map 或统计人类预训练 loss。这样方法才不是黑盒。

5. 实验与结果

5.1 评估任务

论文使用 5 个 dexterous manipulation post-training tasks 检验真实机器人性能，并关注 task completion 和 success rate。任务强调长程、精细和高 DoF 手控制。

实验结果需要和论文主张一一对应：主结果说明系统可用，消融说明设计必要，真实部署说明闭环成立，失败案例则揭示外推边界。

6. 复现要点

6.1 最小复现

完整 20k 小时预训练成本很高。更现实路线是使用公开 checkpoint 或较小人类数据子集，复现 scaling 曲线趋势，再用少量 aligned human-robot data 做目标手 mid-training。
