4.1 总体方法流程

高层 planner 接收自由指令与场景图像，依次执行子任务提议、目标 bounding box 预测、抓取结果识别与任务完成检查。

方法链路可以理解为 VLM planner 产生目标和阶段判断，SAM 将目标框转成 mask，tracker 在执行中维持目标身份，DINOv2 提供冻结视觉特征，diffusion controller 输出 action chunk。每个模块都承担一个明确瓶颈，组合起来减少了训练数据不足时的系统塌陷。

7. 分析、局限与边界

7.1 最有价值的贡献

模块化结构使错误来源可定位，但也产生明显级联风险：VLM 选错物体、SAM 分错 mask 或 tracker 丢失目标，低层控制再强也无法恢复。

它的边界也来自分层假设：上层 grounding 错了，低层再强也只会围绕错误目标执行；mask 丢失或目标身份漂移会直接污染控制条件；低层 diffusion policy 的训练分布仍决定真实接触动作的上限。

这也是论文边界的一部分：它说明当前证据能支撑什么，不能支撑什么。对后续研究来说，最有价值的切入点往往不是照搬整套系统，而是挑出这个瓶颈重新设计数据、表示或评估。

组会讨论时可以把它转成一个追问：如果换成更难的物体、更长的 horizon、更少的数据或另一种手型，论文的哪个假设会最先失效。这个问题通常比单纯复述成功率更能暴露方法的真实边界。

7.2 证据为何站得住

实验主要围绕桌面抓取、清理和先推后抓，尚未证明工具使用、双手协作、透明/强反光物体或精细力控能力。
