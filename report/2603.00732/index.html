1. 论文速览

1.1 UniHM 到底统一了什么

UniHM 面向自由语言指令下的灵巧手操作。它不是简单地让 VLM 输出关节角，而是把不同形态手的人手交互轨迹先压缩成共享离散 token，再让语言模型依据文本、点云与历史动作生成 token 序列，最后结合目标手形态和物理约束解码为可执行轨迹。统一发生在动作语义层，而实际执行仍由具体手型的解码与优化负责。

UniHM 的关键是把语言条件、点云感知、跨手型 motion token 和物理细化接成完整链路。它不是只做抓取姿态，而是生成一段可以被不同手型解码和优化的操作轨迹。

4. 方法详解

4.1 三阶段总体流程

阶段一把不同人手或机器人手轨迹 retarget 到规范表示，并由统一 tokenizer 编码为离散 motion token。阶段二将文本指令、RGB-D 点云和历史 token 输入生成模型，预测后续操作 token。阶段三通过形态相关解码器恢复目标手轨迹，再使用平滑与接触先验进行动态细化。

方法要按三段看：先把轨迹离散成共享 token，再由语言和点云条件生成 token 序列，最后通过目标手型 decoder 与物理优化恢复可执行动作。f(d) 接触核、cold-code 重置和 masked training 都服务于这条链路的稳定性。

从实现角度看，这个模块的输入、输出和误差传播最重要。它一旦出错，下游通常不会自动修复，只会把误差表现成定位失败、轨迹不稳、接触不合理或长程任务中断。

所以方法部分应当抓住三件事：模块接收什么条件，内部把信息压成什么表示，最后怎样变成可执行动作或可评估未来。这样读，论文里的架构图、损失函数和训练阶段就能形成同一条因果链。

8. 组会问答准备

Q1：UniHM 的“统一”体现在哪里？
